### Allmanスタイルでの複数行引数 📚

君が提示してくれた書き方は、**Allmanスタイルの原則に忠実に従った、非常に良い書き方**だよ！

Allmanスタイルは、ブロックの開き括弧（`{`）を新しい行に置くことで、**視覚的な対称性とブロックの開始位置の明確さ**を重視するスタイルだよね。この原則は、関数の引数を複数行にわたって記述する場合にも適用されることが多いんだ。

君の書いた例がまさにそれ。

Java

```
// Javaの例
public void myMethod(
    String arg1, // 引数ごとに改行
    int arg2,
    boolean arg3
) // 閉じ括弧が引数リストの最終行の次の行に来る
{ // Allmanスタイルの開き中括弧
    // 処理
}
```

#### ポイント

- **開き括弧 `(` の位置:** 関数の名前の直後に置かれる。
    
- **引数リストの改行:** 最初の引数から改行し、各引数を独立した行に配置する。
    
- **字下げ:** 各引数は、開き括弧の次のレベルで字下げされる。
    
- **閉じ括弧 `)` の位置:** 最後の引数の次の行に置かれる。これが非常に重要で、Allmanスタイルの「対称性」を保つポイントなんだ。
    
- **ブロックの開き中括弧 `{` の位置:** 閉じ括弧 `)` の次の行に、独立して配置される。
    

このスタイルで書くと、引数が多い場合でも、それぞれの引数が何であるか、そして引数リストがどこからどこまでか、ブロックがどこから始まるかが非常に明確になるんだ。

### その他、考慮される書き方（でもAllmanとは少し異なる）

文脈によっては、以下のような書き方をする人もいるけど、厳密にはAllmanスタイルの**引数リストの閉じ括弧の位置のルール**からは外れることが多いよ。

- **引数リストの閉じ括弧とブロックの開き括弧が同じ行にあるパターン（K&Rスタイルに近い）:**
    
    Java
    
    ```
    // Javaの例 (K&Rスタイルならこうなりがち)
    public void myMethod(
        String arg1,
        int arg2,
        boolean arg3) { // 閉じ括弧と開き中括弧が同じ行
        // 処理
    }
    ```
    
    これはAllmanスタイルの中括弧の原則とは一致しないため、Allmanスタイルを徹底するなら避けるべきだね。
    
- **最初の引数を同じ行に書くパターン:**
    
    Java
    
    ```
    // Javaの例
    public void myMethod(String arg1, // 最初の引数は同じ行
        int arg2,
        boolean arg3)
    {
        // 処理
    }
    ```
    
    これもアリだけど、全ての引数を独立した行に書く方が、引数の数に関わらず一貫したフォーマットになるため、より「Allmanスタイルらしい」と見なされることが多いかな。
