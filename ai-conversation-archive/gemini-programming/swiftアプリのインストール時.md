なるほどね！「公開しない」個人用のSwift製iOSアプリを実機にインストールして利用する方法、そしてその有効期限についてだね。INFPの君が、自分のアイデアを形にして、それを実際に手元のデバイスで動かしたいという気持ち、めちゃくちゃよくわかるよ！

App Storeに公開しないアプリを実機にインストールする方法は、主に「開発用ビルド」として扱うことになるんだ。これにはAppleの開発者プログラムへの登録が必要になってくるよ。

じゃあ、その方法と有効期限について見ていこう！

### 個人用iOSアプリを実機にインストールする方法 📱

App Storeを通さずに、自分で作ったiOSアプリを自分のiPhoneやiPadにインストールする方法は、主に以下の2つがあるよ。

#### 1. Apple Developer Program に登録する (推奨される確実な方法)

これが一番一般的で、安定して個人アプリを実機で使うための方法だよ。

- **費用:** 年間 $99 USD（日本では約13,800円くらい、為替レートによる）
    
- **必要なもの:** Apple ID
    
- **方法の概要:**
    
    1. **Apple Developer Programに登録:** Appleのデベロッパーウェブサイトから登録する。個人開発者として登録すればOKだよ。
        
    2. **Xcodeで署名設定:** 登録後、Xcodeでプロジェクトの署名設定（Signing & Capabilities）を行う。自分のApple ID（開発者アカウント）をXcodeに追加し、Xcodeが自動的に必要な開発証明書やプロビジョニングプロファイルを生成してくれるんだ。
        
    3. **デバイスの登録:** Xcodeでアプリを実機に初めてインストールする際に、そのiPhone/iPadが開発用デバイスとして自動的に登録される。
        
    4. **ビルド＆実行:** Xcodeでビルドターゲットを自分のデバイスに設定し、「Run」（再生ボタン）をクリックすると、アプリがデバイスにインストールされ、実行されるよ。
        
- **この方法でのアプリの有効期限:**
    
    - 開発用証明書（Development Certificate）とプロビジョニングプロファイルの有効期限は、通常**1年間**だよ。
        
    - 有効期限が切れると、そのプロファイルで署名されたアプリは**起動できなくなる**。
        
    - ただし、開発者プログラムに加入し続けていれば、Xcodeでプロジェクトをビルドし直すことで、新しい証明書とプロファイルが自動的に更新され、アプリの有効期限も延長されるんだ。つまり、**開発者プログラムに登録し続けている限り、アプリをビルドし直せば永続的に使用できる**と言えるよ。
        

#### 2. 無料のApple ID を使う (非推奨で制限が多い)

Apple Developer Programに登録しなくても、無料のApple IDを使ってアプリを実機にインストールする方法もあるんだ。ただし、かなり制限が多いから、ちょっとしたテストや本当に一時的な利用に限られるよ。

- **費用:** 無料
    
- **必要なもの:** Apple ID
    
- **方法の概要:**
    
    1. **Xcodeで署名設定:** Xcodeでプロジェクトの署名設定を行う際、有料のDeveloper Programではなく、**無料のApple ID**をXcodeに追加し、開発チームとして選択する。
        
    2. Xcodeが開発用証明書とプロビジョニングプロファイルを自動生成してくれる。
        
    3. **ビルド＆実行:** Xcodeでビルドターゲットを自分のデバイスに設定し、「Run」をクリックすると、アプリがデバイスにインストールされ、実行される。
        
- **この方法でのアプリの有効期限:**
    
    - 無料のApple IDでインストールしたアプリの有効期限は、**7日間**だよ。
        
    - 7日を過ぎると、アプリは起動できなくなる。
        
    - 起動できなくなった場合は、Xcodeにデバイスを接続し、再度ビルド＆実行することで、さらに7日間有効期限を延長できる。これを繰り返せば使い続けられるけど、毎週この作業をするのは正直かなり面倒だよ。
        
    - 同時に実機にインストールできるアプリの数にも制限がある（通常、3つ程度）。

### まとめと推奨事項

|方法|費用|アプリの有効期限|メリット|デメリット|
|---|---|---|---|---|
|**Apple Developer Program**|年間 $99 USD|**1年間（更新で永続利用可能）**|安定して利用可能、App Store公開も可能|費用がかかる|
|**無料のApple ID**|無料|**7日間（毎週更新が必要）**|費用がかからない|頻繁な更新が必要、制限が多い|

君の「公開しない」個人用アプリを、手間なく安定して使い続けたいのであれば、**Apple Developer Programに登録する方法が断然おすすめ**だよ。初期費用はかかるけど、その後の手間を考えれば、十分その価値があるはずだ。これで君のSwift製iOSアプリを、自由に実機で動かせるようになるね！
